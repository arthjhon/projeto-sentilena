01 — Visão Geral

O que é o Sentinela?

Sistema completo de telemetria ambiental com hardware embarcado (ESP32 + sensores), comunicação via MQTT sobre WebSocket, backend Supabase e painel web React. Dados de pH, temperatura e turbidez coletados a cada 5 segundos, disponíveis em tempo real.

Monitoramento Contínuo

Bóias flutuantes com sensores de qualidade da água transmitem dados a cada 5 segundos via MQTT para o broker HiveMQ Cloud.

Painel Administrativo

Dashboard com telemetria em tempo real, gerenciamento de bóias e sensores, e controle de usuários com autenticação Supabase.

Mapa Interativo

Digital twin das lagoas com heatmap de qualidade, slider de histórico temporal e dados ao vivo sobreposto em mapa real.

OTA Remoto

Atualização de firmware dos dispositivos ESP32 sem precisar ir ao campo. Upload via painel, distribuição por MQTT + Supabase Storage.

Alertas Ecológicos

Detecção automática de desvios críticos (pH fora da faixa, turbidez alta) para ação rápida das equipes de campo.

Multi-usuário

Controle de acesso por papel (admin / operador). Administradores acessam OTA e gerência de usuários; operadores monitoram dados.

04 — Firmware

ESP32 — FreeRTOS + OTA

Firmware modular com três tasks paralelas de leitura, mutex para acesso seguro aos dados e suporte a atualizações remotas via HTTP.

Tasks FreeRTOS

  • vTaskReadTurbidity — sensor analógico, cálculo NTU por tensão
  • vTaskReadPH — média circular de 10 amostras, calibração linear
  • vTaskReadTemperature — DS18B20 via OneWire, resolução 10-bit
  • Mutex compartilhado protege acesso às variáveis globais
  • Publicação a cada 5 s via PubSubClient (TLS 8883)

Fluxo OTA

  • Callback MQTT seta flag otaPending = true
  • Loop principal verifica flag antes de publicar sensores
  • HTTPUpdate baixa .bin da URL do Supabase Storage
  • Callbacks de progresso publicam status no tópico OTA
  • Após flash: publica "success" → ESP.restart()
// Loop principal — publicação a cada 5 s
void loop() {
  if (!mqttClient.connected()) reconnectMQTT();
  mqttClient.loop(); // processa ota/command

  if (otaPending) {
    otaPending = false;
    performOTA(otaUrl, otaVersion); // pode reiniciar
    return;
  }

  // Lê sensores com mutex e publica JSON
  if (xSemaphoreTake(xSensorDataMutex, 100ms)) {
    mqttClient.publish("esp_sururu/sensores", payload);
    mqttClient.publish("esp_sururu/status", status);
  }
  delay(5000);
}

Novos Sensores + Conectividade 4G

  • Módulo SIM7600 (4G/LTE) — elimina dependência de Wi-Fi; bóia opera em campo aberto via rede celular
  • Sensor de OD (Oxigênio Dissolvido) — Atlas Scientific EZO-DO; indicador crítico para saúde aquática
  • Sensor de Salinidade / Condutividade — EZO-EC; essencial para lagoas com influência marinha
  • Sensor de Nível d'água — variação do lençol e maré
  • Salvar leituras no Supabase + histórico por data
  • Sistema de alertas por threshold com notificação por e-mail
◎ Futuro — Case + Escala

Modelagem da Case + Expansão

  • Design da case protetora — modelagem 3D (Fusion 360 / SolidWorks) de invólucro impermeável IP67 com encaixe para sensores e antena 4G
  • Fabricação em PETG ou ABS — impressão 3D + vedação com o-rings; flutuador integrado à estrutura
  • Painel solar + bateria LiPo para operação autônoma 24/7
  • Grafana embeddado com time-series InfluxDB
  • Painéis públicos com dados abertos para pesquisadores
  • Predição de qualidade da água por ML
  • Expansão da frota: SM-02, MG-01 e novas bóias
